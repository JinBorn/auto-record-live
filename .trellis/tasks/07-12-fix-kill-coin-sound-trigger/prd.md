# 修复主播被击杀时误播金币音效

## Goal

确保金币掉落音效只用于可确认的主播击杀事件，不在主播死亡或击杀/死亡混合变化的事件上误播。

## Confirmed Facts

- `session-20260617073649-4b5ec478` 的 match 02 成片约 `03:43` 播放了金币音效。
- 该音效映射到源视频 `594.000s` 的 KDA 事件：`kills=2->3 deaths=0->1 previous_at=560.000 current_at=594.000`。
- 当前编辑器仅检查击杀数是否增加，因此将上述同时增加击杀和死亡的混合 KDA 事件当成主播击杀。
- 现有测试只覆盖了 `kills` 不变、`deaths` 增加的纯死亡事件，没有覆盖击杀和死亡同时增加的场景。

## Requirements

- 金币掉落音效只由 `kills` 增加且 `deaths` 不增加的 KDA 事件触发。
- `deaths` 增加时，即使同一 OCR 观察区间内 `kills` 也增加，也不触发金币音效。
- 保持纯击杀、多杀音效选择和时间轴映射行为不变。
- 增加回归测试覆盖击杀和死亡同时增加的 KDA 事件。

## Acceptance Criteria

- [x] `kills=2->3 deaths=0->1` 不生成金币或多杀音效。
- [x] `kills=2->3 deaths=0->0` 仍生成一次金币音效。
- [x] `kills=2->4 deaths=0->0` 仍可生成多杀音效。
- [x] 相关编辑器测试通过。

## Out of Scope

- 重新识别或重建历史视频中的精确击杀时刻。
- 修改高光窗口生成、KDA OCR 采样频率或已有导出视频。
