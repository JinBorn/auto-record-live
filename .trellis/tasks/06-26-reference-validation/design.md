# Technical Design: End-to-End Reference Validation

## Overview

Add final validation coverage around the completed demo-inspired editing
package. The validation should exercise cross-stage contracts without requiring
large Bilibili reference binaries or slow full-video renders.

The task should prefer tests that assert durable artifacts and ffmpeg command
shape. A real media smoke test may be added only if it stays tiny, deterministic,
and skipped when local media dependencies are unavailable.

## Validation Layers

### 1. Contract Integration Test

Create a small fixture in a temporary storage root:

- `match-boundaries.jsonl` with one complete match boundary.
- `subtitle-assets.jsonl` with a real SRT containing high-signal Chinese/English
  cues.
- `highlight-plans.jsonl` either generated by `HighlightPlannerService` or
  seeded with a valid highlight window.
- optional fake `recording-assets.jsonl` and `export-assets.jsonl` rows so
  `CopywriterService` can produce publishing package metadata.

Then run the real planner/copy services that do not require expensive media:

```text
HighlightPlannerService -> EditingPlannerService -> CopywriterService
```

Assertions:

- highlight plan exists and contains high-signal windows.
- edit plan contains teaser segment(s) followed by exactly one full `main`
  segment.
- no timeline segment has `role="insert"` or a non-null `source_path`.
- publishing package evidence and cover lines prefer the highlight cue rather
  than only the first subtitle line.
- rerunning remains idempotent where each stage promises idempotency.

### 2. Combined Exporter Command Regression

Extend exporter tests with one combined command-path case:

- `settings.export.use_edit_plans=True`
- `settings.export.burn_subtitles=True`
- `settings.export.use_ass_subtitles=True`
- edit plan has:
  - teaser segment with `TimelineVideoTransform(kind="punch_in")`
  - full main segment
  - one BGM `AudioBed`
  - one `SoundEffectHit`

Patch ffmpeg execution as existing exporter tests do. Assert the command:

- uses `-filter_complex`
- references a generated `.ass` subtitle sidecar
- includes per-segment `trim`/`atrim` and concat in teaser-before-main order
- includes `scale`/`crop` punch-in filters on the teaser chain
- includes BGM/SFX input paths and shifted audio filter labels
- includes `amix=duration=first`
- does not use the older highlight `select=` path

This test catches cross-feature collisions without doing a real render.

### 3. Optional Tiny FFmpeg Smoke

If implemented, place it in a focused test that skips when `ffmpeg` or
`ffprobe` is unavailable. It should:

- generate an 8-12 second synthetic MP4 using `lavfi` `testsrc` + `sine`
- write a real SRT with one short cue
- seed a minimal edit plan
- run `ExporterService` with ffmpeg enabled
- assert the output file exists, probes as a video stream, and has non-zero
  duration

Keep this smoke test independent from `data/demo1` and `data/demo2`.

## Manual Reference Checklist

Add a task-local checklist, for example:

```text
.trellis/tasks/06-26-reference-validation/manual-reference-checklist.md
```

The checklist should map demo observations to current project checks:

- cover/copy: title, summary, cover lines, evidence, optional cover path
- teaser: teaser segment(s) before full main segment
- subtitles: ASS sidecar style and burn-in path
- music/SFX: low gain values and explicit local paths
- zoom: punch-in only on selected high-signal teaser
- removed scope: external reference inserts are intentionally not validated

## Data Boundaries

- `data/demo1` and `data/demo2` remain local human references only.
- New automated fixtures must live under test temp directories.
- No test should commit, copy, or mutate demo media.
- Durable JSONL contracts should use existing shared models and
  `append_model`/`load_models`.

## Compatibility

- No production behavior change is required unless validation reveals a real
  integration bug.
- Existing tests should remain the primary feature-specific safety net.
- New validation should not weaken current command assertions.

## Rollback

- If the optional smoke test is flaky, keep the contract and command tests and
  remove or skip the real media smoke.
- If validation reveals a bug, fix it in the owning stage and update the
  corresponding focused test, then keep the integration test as the regression.
